# 작업 진행상황

## 진행상황
- 인라인 링크는 primary 컬러를 명시적으로 사용하도록 `text-[var(--theme)]` 스타일로 조정했습니다.
- Markdown 경량 파서의 리스트 인식 로직을 보완했습니다.
  - 라인 앞 공백(들여쓰기)이 있어도 리스트로 인식
  - `- item` 뿐 아니라 `* item` 불릿도 인식
  - 문단 파싱 중 다음 라인이 리스트/헤딩/코드블록 시작인지 판별할 때 `trimStart()` 기준으로 체크
- 위 변경으로 화면에서 불릿이 빠지던 케이스(앞 공백이 있는 `- ...` 라인)도 렌더링되도록 개선했습니다.

## Next Action Item
- 실제 콘텐츠 파일 2~3개에서 아래 케이스를 추가 확인
  - 공백 + `-` 불릿
  - `*` 불릿
  - 번호 리스트 뒤에 이어지는 불릿 리스트
- 필요 시 중첩 리스트(2-depth 이상) 렌더링 규칙 확장

## 주의사항
- 현재 파서는 커스텀 경량 파서이므로 CommonMark/GFM 전체 스펙을 100% 지원하지는 않습니다.
- 복잡한 중첩 리스트/체크리스트/테이블은 추가 구현이 필요합니다.

## 사용자가 해야할 일
1. `pnpm install`
2. `pnpm build`
3. `pnpm dev`
4. 문제였던 포스트 화면에서 확인
   - 링크 텍스트가 primary 컬러로 보이는지
   - 기존에 불릿이 깨지던 문장이 불릿으로 표시되는지
