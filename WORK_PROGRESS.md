# 작업 진행상황

## 진행상황
- 댓글(utterances) 영역이 실제로 로딩되는지 사용자에게 보이도록 개선했습니다.
  - 위젯 로딩 상태 문구(`댓글 위젯을 불러오는 중...`)를 추가
  - DOM 변화를 감지해 `iframe.utterances-frame`이 삽입되면 로딩 문구를 자동 제거
  - 댓글 컨테이너에 최소 높이를 부여해 렌더 전에도 영역이 보이게 처리
- 마크다운 리스트 파서를 개선해 들여쓰기 구조를 더 자연스럽게 렌더링하도록 반영했습니다.
  - ordered list 내부의 들여쓴 불릿(`-`, `*`)을 하위 리스트로 인식
  - 들여쓴 연속 텍스트를 직전 항목 문장/하위 항목에 이어 붙이도록 처리
  - 이로 인해 `1. ...` 다음에 `2. ...`가 끊기지 않고 같은 `<ol>`에서 정상 증가
- 기존 이미지/캡션 처리(`![alt](src "caption")`)와 링크 primary 컬러 적용은 유지했습니다.

## Next Action Item
- 중첩 리스트 2-depth 이상(ordered > unordered > ordered) 케이스 렌더링 정책 확장
- 필요 시 이미지를 block 전용으로 강제 분리(문단 중간 이미지 정렬 정책)
- utterances 설정(repo/theme/label/issue-term)을 환경변수화할지 결정

## 주의사항
- 현재 파서는 커스텀 경량 파서라 CommonMark/GFM 전체를 완전 지원하지 않습니다.
- utterances는 외부 스크립트 로드 의존이 있으므로, 네트워크 상태/브라우저 차단 정책에 영향을 받을 수 있습니다.

## 사용자가 해야할 일
1. `pnpm install`
2. `pnpm build`
3. `pnpm dev`
4. 포스트 상세 페이지에서 확인
   - `1.` 다음이 `2.`로 정상 증가하는지
   - 들여쓴 `-` 항목이 하위 불릿으로 렌더링되는지
   - 하단 `댓글` 섹션에서 로딩 문구 이후 utterances가 표시되는지
