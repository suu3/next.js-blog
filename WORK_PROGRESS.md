# 작업 진행상황 (상세 페이지 파싱/TOC + 그림자 두께 조정)

## 진행상황
- 사용자 피드백 2가지를 우선 반영했습니다.
  1. 그림자 두께가 너무 굵은 문제
  2. 게시글 상세(`/posts/...`)에서 본문 파싱/사이드바(TOC) 미노출 문제
- 그림자 두께 조정
  - 전역 프레임/히어로/GNB 그림자 오프셋을 한 단계 축소
  - 상세 본문 카드/TOC 카드 그림자도 `2px` 수준으로 완화
- 상세 본문 렌더링 개선
  - 경량 마크다운 파서를 개선해 아래 블록 렌더링 지원
    - `##`, `###` 제목
    - 일반 문단
    - 순서 없는 목록(`-`)
    - 순서 있는 목록(`1.`)
    - fenced code block
  - 인라인 문법 일부(`**bold**`, `` `code` ``, `[link](url)`)도 렌더링되도록 보완
- TOC(글 제목 연동 사이드바) 복구
  - `##`, `###`에서 heading ID를 생성
  - 우측 sticky 사이드바에 Index 렌더링
  - 스크롤 위치에 따라 active 항목이 강조되는 클라이언트 TOC 컴포넌트 추가

## Next Action Item
- 현재 경량 파서 대신 remark/rehype 기반으로 확장해 마크다운 호환성 강화
- 모바일에서 TOC를 접이식 패널로 제공할지 UX 검토
- 상세 본문 코드블록 문법 하이라이팅 추가 여부 결정

## 주의사항
- `dangerouslySetInnerHTML`는 로컬 컨텐츠 마크다운 렌더링 목적에서만 사용합니다.
- 복잡한 마크다운(테이블/중첩 리스트/각주)은 아직 완전 대응하지 않습니다.

## 사용자가 해야할 일
- 아래 순서로 확인해주세요.
  1. `pnpm install`
  2. `pnpm build`
  3. `pnpm dev`
  4. 문제 URL 상세 페이지에서
     - 본문이 코드덩어리처럼 깨지지 않는지
     - 우측 Index(TOC)가 보이는지
     - 스크롤 시 TOC 활성 항목이 바뀌는지 확인
